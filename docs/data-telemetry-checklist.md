# Data & Telemetry Checklist

Purpose
- Ensure features are instrumented with actionable telemetry and success metrics are measurable.

Checklist
- [ ] Product Analyst / Data Analyst assigned
- [ ] Success metrics defined in the issue or one-pager
- [ ] Telemetry events and schemas specified
- [ ] Tests/validations for telemetry in place where possible
- [ ] Dashboards and alerts created for key signals
- [ ] Plan for data retention and privacy considerations reviewed

Notes
- Link dashboards and example queries in the issue so reviewers can validate signal coverage during review.
