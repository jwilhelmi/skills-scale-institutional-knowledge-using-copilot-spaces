# OctoAcme Project Management Documentation

Welcome to the OctoAcme Project Management documentation hub. This collection provides guidance and best practices for running successful projects across our organization.

## About OctoAcme's Approach

OctoAcme's project management methodology is built on delivering customer value through structured, iterative processes. We balance the rigor of comprehensive planning with the flexibility needed for modern software delivery. Our approach emphasizes clear ownership, data-driven decisions, and continuous improvement.

### What We Do

OctoAcme manages the complete project lifecycle—from initial problem validation through deployment and retrospectives. Our processes ensure:

- **Clear stakeholder alignment** before work begins
- **Well-scoped, incremental delivery** with defined acceptance criteria
- **Transparent tracking** of progress, risks, and dependencies
- **Quality gates** at each stage (planning, execution, release)
- **Continuous learning** through structured retrospectives and process improvements

Each project is stewarded by a **Project Manager** (who coordinates delivery and timeline) and a **Product Manager** (who defines outcomes and measures success), supported by a cross-functional delivery team.

## Core Principles

OctoAcme projects are built on five core principles:

- **Customer-first**: Prioritize customer value and usability
- **Iterative delivery**: Deliver small, testable increments
- **Clear ownership**: Each project has named owners and defined roles
- **Data-informed decisions**: Measure impact and iterate based on evidence
- **Psychological safety**: Encourage feedback and learning

---

## Key Roles

Our projects typically involve these roles:

- **Project Manager (PM)**: Coordinates delivery, schedules, risk, communications
- **Product Manager (PdM)**: Defines outcomes, prioritizes backlog, measures success
- **Developers**: Implement features and collaborate on design
- **QA/Testing**: Validates quality and acceptance criteria
- **Stakeholders**: Provide inputs and approvals

See [Roles & Personas](octoacme-roles-and-personas.md) for detailed descriptions.

## Project Lifecycle Overview

### 1. **Initiation** → Define the Problem, Align Stakeholders
Validate business need, identify success metrics, confirm team availability. Outputs: Project One-pager, stakeholder alignment, go/no-go decision.

### 2. **Planning** → Create an Actionable Plan
Break work into shippable increments, estimate scope, define acceptance criteria, identify dependencies. Outputs: Prioritized backlog, release plan, risk register.

### 3. **Execution** → Build, Test, Track Progress
Daily standups, sprint execution, code reviews, quality validation. Outputs: Completed features, passing tests, demo artifacts.

### 4. **Release** → Deploy & Verify
Pre-release checks, deployment to staging and production, smoke tests, announcement. Outputs: Deployed feature, release notes, verified functionality.

### 5. **Close & Learn** → Capture Learnings
Retrospective to identify what went well and opportunities for improvement. Outputs: Documented learnings, actionable improvements added to backlog.

## Communication Cadence

Our projects follow a standard communication rhythm:

- **Daily standups** (15 min): Progress, blockers, dependencies
- **Weekly PM sync**: Delivery coordination and risk review
- **Weekly stakeholder update**: Status, milestones, risks, decisions needed
- **Sprint/iteration review**: Demo and acceptance
- **Retrospective**: Post-sprint or post-release learnings
